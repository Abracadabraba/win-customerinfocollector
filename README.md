# 制药用水设备展会客户收集（Windows 桌面版）

根据《客户信息登记表-SE.docx》开发的客户信息采集工具，Windows 桌面版，支持：

1. 展会/客户基本信息填写；展会名称、日期、销售人员均可从"设置"里预先配置，新建记录时自动带出、下拉快选
2. 姓名前可选称谓（Mr. / Ms. / Mrs. / Dr. / Eng. / 其他手动输入）
3. 按所选产品自动展开对应的问询表单（可多选，每个产品均带"备注"，防止遗漏未在选项中的内容）；电压、频率、相数三项在同一行选择，电压/频率均为下拉菜单（电压支持"其他"手动输入），从第二台设备开始会自动带入第一台设备的电压/频率/相数/认证要求，可一键"重新复制"或直接改成不同内容（设备认证要求始终可单独修改）
4. 填完产品需求后填写 "GMP Compliance & Process Requirements"
5. 选择沟通人（"设置"里维护的名单 + 其他手动输入），同一份名单同时用于 Sales Rep 下拉菜单
6. 本地保存所有客户记录；后续如有新增需求，可随时调出记录再编辑，
   每次编辑保存都会生成新版本号 R1、R2、R3……（改几次就是 R 几），历史版本会保留在文档的"修改记录"中
7. 一键导出为 Word (.docx) 文档
8. 国家下拉菜单（按中文拼音排序，可搜索），选择国家后自动带出电话区号
9. 职位下拉菜单（Owner / Director / Purchasing Manager / Sales / Engineer / 其他手动输入）
10. 拍摄名片：调用电脑摄像头拍照，自动尝试识别电话、邮箱、网址并填入对应字段；识别出的文字也会列出来，点一下就能填入"姓名"或"公司"；名片照片会跟着这条客户记录一起保存，导出 Word 时会插入到客户信息下方；照片方向会自动转正为横向，转得不对也可以手动点"旋转90°"
11. 表面处理（Ra）做成下拉菜单：0.4 / 0.6 / 0.8 / 其他手动输入
12. 导出的 Word 文档会自动保存进电脑的 **下载（Downloads）** 文件夹，路径为
    `下载/<展会名称>/<月日 如 0730>/<国家_客户名_产品名称_日期>.docx`
13. 首页底部和导出的 Word 文档末尾/页脚都印有 "© SKE&Eagle Group" 版权声明

数据保存在电脑本地（不联网、不上传服务器），卸载程序或清除数据会导致记录丢失，如需要请及时导出 Word 备份。

## 关于名片扫描识别（重要说明）

- 拍照用的是电脑摄像头，文字识别用的是 Tesseract.js（开源离线识别引擎，支持中英文），全程不上传任何图片到服务器。
- **电话、邮箱、网址** 这三项因为有明显的格式规律（如包含 @、http、连续数字），识别准确率较高，会自动尝试填入对应字段。
- **姓名、公司** 也会做自动猜测（公司名靠"Ltd/Co./Inc./Group"等关键词判断，姓名靠"2-4个单词、不含数字"判断），猜中了会自动填入空白字段；猜错了或者已经手动填过，可以点下方"识别出的文字"列表里正确的那一行，重新填入"姓名"或"公司"。
- 已经做了两项针对名片这种"左右两栏排版"的优化：让识别引擎按"零散文字块"而不是"一整段文章"的方式去读（避免把左栏和右栏的文字拼到同一行），并且在识别前对照片做了灰度+对比度增强处理。整体识别效果会比"随手拍一张照片直接识别"好不少，但离"扫描仪级别的100%准确"还是有差距，纯文字类的姓名、公司名建议识别完随手检查一下。
- **第一次**使用识别功能时需要联网下载一次识别语言包（几MB到几十MB，只有第一次要联网，下载完会被缓存，之后可以离线用）。如果网络不好导致下载失败/超时，换个网络环境重试一次即可。
- 如果拍完之后完全没有任何文字被识别出来，会提示"未识别到任何文字"，这种情况通常是照片模糊/光线不足，建议重新拍摄，光线充足、正对名片、尽量占满画面。

## 关于保存 / 导出 / 返回的逻辑（重要说明）

- **保存**：把当前填写的内容存入客户列表。第一次点"保存"会新建 1 条记录；之后再次点"保存"（无论是刚保存完接着导出，还是过后再编辑），都只会更新同一条记录，不会重复出现多条。
- **导出 Word 文档**：只是把当前屏幕上的内容渲染成 Word 文件导出到电脑，不会往列表里新增或修改记录，可以重复点导出很多次也不会多出记录。
- **返回首页**：单纯返回客户列表页面，不会自动保存——如果有修改还没点"保存"，回到首页后这些修改不会被保留，请先点"保存"再返回。

## 关于自动保存到"下载"文件夹（重要说明）

- 通过 Electron 桌面程序自带的能力，直接写入系统的"下载(Downloads)"文件夹，不需要任何额外授权。
- 导出成功后会自动弹出文件资源管理器，帮你定位到刚生成的那个文件。

## 关于摄像头打不开 / 画面空白纯色

已经做了两处修复：
- 关闭了可能导致摄像头画面变成纯色/花屏的显卡硬件加速（一个已知的 Electron/Chromium 兼容性问题）。
- 拍名片弹窗里加了"选择摄像头"下拉菜单——如果电脑上装过 OBS、Zoom、腾讯会议等软件，它们有时会注册一个"虚拟摄像头"并被系统排在最前面，导致打开的其实不是真实摄像头。如果画面空白，先在这个下拉菜单里换一个摄像头试试。

如果换了以后还是不行，请把摄像头型号、电脑品牌型号告诉我，我再针对性排查。

## 二、打包成 Windows 安装包（通过 GitHub Actions 自动编译，不需要自己装开发环境）

项目里已经包含 `electron/` 桌面程序代码和 `.github/workflows/windows-build.yml` 自动编译脚本，GitHub 会在云端 Windows 机器上帮你编译出安装包，你只需要下载即可。

### 步骤 1：解压 zip

把我导出的 `customer-app.zip` 解压到你电脑上的任意文件夹。

### 步骤 2：在 GitHub 上新建仓库（repository）

1. 打开 https://github.com ，登录你的账号（没有账号先免费注册）。
2. 右上角点击 "+" → "New repository"。
3. 填写仓库名字，例如 `customer-registration-windows`。
4. 选择 "Private"（私有，别人看不到）或 "Public" 都可以。
5. **不要**勾选 "Add a README file" 等初始化选项（因为我们本地已经有文件了）。
6. 点击 "Create repository"。

### 步骤 3：上传项目代码到 GitHub

打开电脑的终端 / 命令行工具，`cd` 进入解压后的项目文件夹，依次执行（把 `你的用户名` 和 `仓库名` 换成实际的）：

```bash
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

如果没装过 Git，先去 https://git-scm.com/downloads 下载安装。
如果不熟悉命令行，也可以用 GitHub Desktop（图形界面工具，https://desktop.github.com ）：
打开 GitHub Desktop → "Add local repository" → 选择解压后的文件夹 → "Publish repository"。

### 步骤 4：查看自动编译进度

1. 代码推送成功后，打开你的 GitHub 仓库网页。
2. 点击顶部的 "Actions" 标签页。
3. 你会看到一个正在运行（黄色圆点）或已完成（绿色勾）的工作流，名字是 "Build Windows Desktop App"。
4. 点进去等待几分钟（通常 3-8 分钟），编译完成后会显示绿色勾 ✔。

### 步骤 5：下载安装包

1. 在编译完成的那次运行页面里，往下拉，找到 "Artifacts" 区域。
2. 点击 `windows-builds` 下载，解压后里面有**两个文件**：
   - `...Setup....exe`：**安装版**，双击后会走安装向导，装到程序列表里，桌面/开始菜单会有图标。
   - `...(portable) ....exe`：**免安装版（绿色版）**，不需要安装，双击直接运行，可以放在 U 盘里带走用；关闭后不会在电脑里留下"已安装的程序"记录，删掉这一个 exe 文件就等于卸载干净了。
   两个版本功能完全一样，按需选一个用即可。
3. 双击运行（Windows 可能会弹出"Windows 已保护你的电脑"提示，这是因为安装包没有购买代码签名证书，属于正常现象，点"更多信息" → "仍要运行"即可）。

### 以后有代码更新怎么办？

以后如果需要改需求、加产品字段等，改完代码后，在项目文件夹里执行：

```bash
git add .
git commit -m "更新说明"
git push
```

推送后 GitHub 会自动重新触发编译，重复"步骤 4、5"下载新的安装包即可。

也可以完全不改代码，直接在 GitHub 仓库网页的 Actions 页面点击 "Build Windows Desktop App" → "Run workflow" 手动重新触发编译。

### 想在自己电脑上直接跑（不通过 GitHub，仅供开发调试）

如果你的电脑已经装了 Node.js，也可以本地直接跑起来看效果（不会生成安装包，只是临时打开一个窗口）：

```bash
npm install
npm run electron:dev
```

## 三、常见问题

**Q: 编译失败怎么办？**
点开失败的红色 ✗ 记录，展开报错的步骤，把报错信息复制给我，我可以帮你分析修复。

**Q: 数据会不会丢？**
所有客户数据保存在电脑本地存储中。建议定期使用"导出 Word"功能，把重要客户资料导出备份保存。

**Q: 表单字段能不能修改/增加？**
可以。所有字段定义都在 `src/data/formSchema.js` 文件里，按同样格式增删字段即可，改完推送到 GitHub 就会自动重新编译。

**Q: 打开摄像头，弹出权限提示怎么办？**
点"允许"即可。如果 Windows 系统设置里"相机隐私权限"关闭了桌面应用访问摄像头，需要去 Windows 设置 → 隐私和安全性 → 相机，打开"允许应用访问你的相机"。

**Q: 导出的文件在哪？**
会自动存到电脑的 `下载(Downloads)` 文件夹里，按展会名称、日期分好文件夹；导出成功后还会自动弹出文件资源管理器帮你定位到那个文件。

**Q: 名片识别一直失败/很慢？**
第一次使用需要联网下载识别语言包，如果网络不好可能会超时失败，换个网络环境重试一次即可；下载成功后之后就不再需要联网了。

**Q: 想换个程序图标？**
图标文件在 `build-resources/icon.png`，换成你自己的方形图片（建议 1024x1024，PNG 格式）覆盖这个文件，推送到 GitHub 重新编译即可，安装版和免安装版会自动用同一个图标。
